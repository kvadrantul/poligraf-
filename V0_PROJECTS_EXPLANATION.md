# Как работают генерации в v0.dev

## Текущая ситуация

Сейчас приложение использует **v0 Model API** (`/v1/chat/completions`), который:
- ✅ Генерирует компоненты по промпту
- ❌ НЕ создает проекты в вашем аккаунте v0.dev
- ❌ НЕ сохраняет историю генераций
- ❌ НЕ позволяет итерировать/править результаты

Каждый запрос = новый независимый компонент.

## Где можно увидеть генерации?

**Сейчас:** Только в вашем Telegram Mini App. Результаты не сохраняются в v0.dev.

## Варианты решения

### Вариант 1: Использовать v0 Platform API (Рекомендуется)

v0.dev имеет **Platform API**, который позволяет:
- Создавать проекты
- Создавать чаты в проектах
- Итерировать и править компоненты
- Сохранять историю

**Документация:** https://v0.app/docs/api/platform/overview

**Преимущества:**
- Все генерации сохраняются в вашем аккаунте v0.dev
- Можно открыть проект в веб-интерфейсе v0.dev и править
- Можно итерировать через API

**Недостатки:**
- Более сложная интеграция
- Нужно управлять проектами и чатами

### Вариант 2: Локальное сохранение истории

Сохранять результаты в localStorage браузера или отправлять на backend.

**Преимущества:**
- Простая реализация
- История доступна в приложении

**Недостатки:**
- Не связана с v0.dev
- Нужно самому управлять хранением

### Вариант 3: Добавить ссылку на v0.dev

После генерации показывать ссылку для открытия в v0.dev (если Platform API поддерживает).

## Что выбрать?

**Для итераций и правок:** Используйте **v0 Platform API** - это правильный способ работать с проектами.

**Для простых генераций:** Текущий Model API подходит, но результаты не сохраняются.

## Следующие шаги

Если хотите работать с проектами, нужно:
1. Изучить v0 Platform API документацию
2. Обновить backend для работы с проектами/чатами
3. Добавить UI для управления проектами в Mini App
