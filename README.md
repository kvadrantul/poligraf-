# Poligraf - Telegram Mini App

Telegram мини-приложение для генерации UI компонентов через v0.dev API.

## Структура проекта

- `index.html` - основной HTML файл приложения
- `styles.css` - стили для интерфейса
- `app.js` - JavaScript логика с интеграцией Telegram Web App API и v0.dev
- `api/generate.js` - backend endpoint для работы с v0.dev API (Vercel Serverless Function)
- `vercel.json` - конфигурация для Vercel

## Функциональность

- ✅ Поле для отображения результата (верхняя часть) - показывает сгенерированный код
- ✅ Текстовое поле для ввода промпта (нижняя часть)
- ✅ Кнопка отправки промпта в v0.dev API
- ✅ Отображение всех результатов в одном проекте (история)
- ✅ Индикатор загрузки при генерации
- ✅ Поддержка тем Telegram (светлая/темная)
- ✅ Адаптивный дизайн
- ✅ Вибро-отклики при взаимодействии

## Как работает

1. Пользователь вводит промпт (например: "Создай кнопку с текстом Привет")
2. Промпт отправляется на backend endpoint
3. Backend отправляет запрос к v0.dev API
4. Результат (код/разметка) отображается в поле результатов
5. Можно отправлять несколько промптов подряд - все результаты сохраняются

## Быстрый старт

### 1. Настройка Frontend (GitHub Pages)

1. Файлы уже задеплоены на GitHub Pages: `https://kvadrantul.github.io/poligraf-/`
2. Подключите к Telegram боту через [@BotFather](https://t.me/BotFather)

### 2. Настройка Backend (Vercel)

**ВАЖНО:** Для работы с v0.dev API нужен backend для безопасного хранения API ключа.

См. подробную инструкцию в [BACKEND_SETUP.md](./BACKEND_SETUP.md)

Кратко:
1. Задеплойте `api/generate.js` на Vercel
2. Добавьте переменную окружения `V0_API_KEY` с вашим API ключом от v0.dev
3. Обновите `API_ENDPOINT` в `app.js` на URL вашего Vercel проекта
4. Закоммитьте и запушьте изменения

### 3. Получение API ключа v0.dev

1. Войдите на [v0.dev](https://v0.dev)
2. Перейдите в настройки: `https://v0.dev/chat/settings/keys`
3. Создайте новый API ключ

## Безопасность

⚠️ **НИКОГДА не добавляйте API ключ в клиентский код!**
- API ключ должен быть только в переменных окружения на сервере
- Клиентский код отправляет запросы только на ваш backend
- Backend использует API ключ для запросов к v0.dev
